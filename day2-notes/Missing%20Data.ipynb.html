Introduction

R has a huge number of options for missing data. In the R overview of data cleaning, 19 distinct packages are avaliable for missing data. Together they cover every concievable approaches to missing values including mean/meadian imputation, LOCF, nearest neighbour, hot deck, random forest, EM, bootstrap EM, and a veriety of multiple imputation procedures.

Today we will focus on the package Amelia II for the following reasons:

  1. It is extremely fast
  2. I have not known it to crash yet (not converge yes but never crash)
  3. Will happily deal with numeric, nominal, or ordinal variables (this is where the speed helps a bunch)
  4. Will impute cross-sectional, time series, cross-sectional time series data
  5. Really useful diagnostics including the ability to over impute observed variables to see if the missing data model is working

Amealia creates multiple imputations by bootstrapping the EM algorithm.

Top tips

  1. Amealia II is an easy to use imputation package
  2. mitools OR Zelig packages provides an easy way of dealing with multiple imputed data
  3. semTools or lavaan.survey will provide means of dealing with multiple imputed data in SEM
  4. In amelia look at the "chain length" i.e. numbers under --Imputation N -- output. If they are roughly similar for each imputation then the MI has worked well.
  5. Working with multiple imputed data requires the use of existing tools AND knowing how to use sapply and lapply
  6. Anything you don't want imputed include in the "idvars" argument in amelia
  7. I have not done so here BUT amelia will be considerably faster by running it in parallel (see help file)

Analysing Imputed Data

There are essentially four ways of analysing your muliple imputation data

  1. mittools using the with command
  2. mittols in combination with the survey package (i.e., What we did for the SEM models)
  3. The Zelig package
  4. User defined functions

I have used all four in the past. Today we will focus on a combination of 1 and 4.

Multilevel Models

Multilevel models are a little harder in that they require a few additional lines of code. Strucutral equation models we covered by way of exploring plausible values in the previous workshop with lavaan or semtools covering all the heavy lifting.

Model2 <- with(pisaMI, lmer(SCMAT ~ ESCS + (1 | SCHOOLID) ) )
