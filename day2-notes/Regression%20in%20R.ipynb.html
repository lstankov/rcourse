Top Tips for running regression

  1. Almost ALL model functions will have the following structure response∼predictor+predictor
  2. Once models are run you can pass them to summary() OR plot() OR residuals() OR coef() OR confint() OR predict() OR lm.beta()
  3. Standardized betas come from the lm.beta() function that is in the QuantPsyc package
  4. Interactions can be defined in models short hand via response∼predictor∗predictor or long hand response∼predictor+predictor+predictor:predictor
  5. You can make ANY transformation you want directly in the model including polynomials, interactions, log transformations, z-scores
  6. To compare the fit of nested models use anova()
  7. To calculate simple slopes use deltaMethod() in the car package
  8. To calculate mediation use the mediation package

Mediation

Mediation is used a lot in the sort of research we do. Here we are going to look at a general purpose approach to mediation that will work regardless of whether your models are regression, multilevel, logistic, or polr. To do this we will use the mediation package. The good thing about mediation is that it will produce estimates via boot strapping or simulation those overcoming some complaints about the Barron and Kenny method.

The question we will ask is the degree to which the effect of gender on SCMAT is mediated by achievement.

%%R
#load the mediation package
library(mediation)
# here we will use the subset command to account for missing data on SCMAT
# First we explore the effect of gender on achievement
modelA <- lm(PV1MATH ~ gender, data = pisa, subset = !is.na(SCMAT) )
# Second we estimate the effect of gender and PV1MATH allowing for a treatment by mediator interaction
modelB <- lm(SCMAT ~ gender*PV1MATH, data = pisa, subset = !is.na(SCMAT) )
# Now we calculate the mediation using simulation (see help file for how to do bootstrapping)
modelAB <- mediate(modelA, modelB, treat = 'gender', mediator = 'PV1MATH', sims = 100)
# Get the results and plot them
print(summary(modelAB) )
plot(modelAB)

Multilevel Models

Top tips for multilevel models

  1. ICC1 and ICC2 functions in the multilevel package are easy ways ti get ICCs
  2. Multilevel models look exactly like lm or glm models except for the addition of a random component
  3. The random component is an additional predictor and looks like (bitYouWantRandom | cluster)
  4. 1 = intercept so a random intercept for school id would look like (1 | schoolId)
  5. This structure is identical for either lm or glm models
  6. Three or more level nesting looks like (1 | classID/schoolID )
  7. Cross classification looks like (1 | schoolID) + (1 | neighbourhoodID)
  8. You can use a hausman endogineity test to see whether you even need a multilevel model
  9. You can compare nested multilevel models using anova()
  10. Multilevel models are run from the lme4 package, however if you need to account for autoregressive errors you will need to use the nlme package

%%R
#First we need to write a function for the HET
phtest_glmer <- function (glmerMod, glmMod, ...)  {  ## changed function call
  coef.wi <- coef(glmMod)
  coef.re <- fixef(glmerMod)  ## changed coef() to fixef() for glmer
  vcov.wi <- vcov(glmMod)
  vcov.re <- vcov(glmerMod)
  names.wi <- names(coef.wi)
  names.re <- names(coef.re)
  coef.h <- names.re[names.re %in% names.wi]
  dbeta <- coef.wi[coef.h] - coef.re[coef.h]
  df <- length(dbeta)
  dvcov <- vcov.re[coef.h, coef.h] - vcov.wi[coef.h, coef.h]
  stat <- abs(t(dbeta) %*% as.matrix(solve(dvcov)) %*% dbeta)  ## added as.matrix()
  pval <- pchisq(stat, df = df, lower.tail = FALSE)
  names(stat) <- "chisq"
  parameter <- df
  names(parameter) <- "df"
  alternative <- "one model is inconsistent"
  res <- list(statistic = stat, p.value = pval, parameter = parameter, 
              method = "Hausman Test",  alternative = alternative,
              data.name=deparse(glmerMod@call$data))  ## changed
  class(res) <- "htest"
  return(res)
}

#plot
visualize.lme <- function (model, coefficient, group, ...){
        r <- ranef(model)[[group]]
        f <- fixef(model)

        effects <- data.frame(r[,1]+f[1], r[,2]+f[2])

        number.lines <- nrow(effects)

        predictor.min <- tapply(model@frame[[coefficient]], model@frame[[group]], min, na.rm=TRUE)
        predictor.max <- tapply(model@frame[[coefficient]], model@frame[[group]], max, na.rm=TRUE)

        outcome.min <- min(predict(model), na.rm=TRUE)
        outcome.max <- max(predict(model), na.rm=TRUE)

        plot (c(min(predictor.min),max(predictor.max)),c(outcome.min,outcome.max), type="n", ...)

        for (i in 1:number.lines){
            expression <- function(x) {effects[i,1] + (effects[i,2] * x) }
            curve(expression, from=predictor.min[i], to=predictor.max[i], add=TRUE)
            }
        }
